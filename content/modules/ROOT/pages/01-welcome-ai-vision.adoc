= Welcome to AI Computer Vision at the Edge
:_content-type: CONCEPT

++++
<span class="workshop-time-badge">⏱ ~5 min</span>
++++

[.lead]
*You are about to deploy AI-powered PPE detection across a distributed edge fleet.*

In the next few hours you will operate a production-grade platform that federates computer-vision inference across multiple Red Hat OpenShift Container Platform (OCP) clusters — detecting hardhats, safety vests, and goggles in real time using YOLO and OpenVINO models served by Red Hat OpenShift AI (RHOAI).

== What you will accomplish

By the end of this workshop you will have:

* Explored a *hub-spoke fleet* managed by Red Hat Advanced Cluster Management for Kubernetes (RHACM)
* Deployed *NeuroFace* — a multimodal AI application that runs PPE safety detection, face recognition, and object detection
* Inspected *KServe InferenceServices* and *OpenVINO Model Server (OVMS)* serving real ML models
* Connected spoke clusters to the hub with *Skupper Service Interconnect*
* Reviewed *security posture* with Red Hat Advanced Cluster Security for Kubernetes (RHACS) and HashiCorp Vault
* Observed cross-cluster traffic through *Service Mesh 3*, *Grafana*, and *Kiali*
* Created a project from a *Developer Hub* software template

== Architecture at a glance

The pattern deploys three clusters in a hub-and-spoke topology:

[cols="1,2,3",options="header"]
|===
| Cluster | Role | Highlights

| *Hub*
| Fleet control plane
| RHACM, Vault, RHACS Central, Developer Hub, GitLab, OpenShift AI, Quay, Keycloak, NeuroFace gateway

| *East spoke*
| Edge inference
| NeuroFace, YOLO PPE, OVMS, Skupper, Service Mesh ambient, observability stack

| *West spoke*
| Edge inference
| Same as east — traffic load-balanced 50/50 via the hub Gateway API HTTPRoute
|===
