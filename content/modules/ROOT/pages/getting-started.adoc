= Getting started
:_content-type: PROCEDURE

Use this procedure to verify your workshop environment and access key platform services.

== Logging in to the hub cluster

. Open the OCP Console from the cluster application menu.
. Select the `workshop-users` identity provider.
. Log in with your assigned username (for example, `user1`) and password `Welcome123!`.

Alternatively, use the embedded terminal:

[source,sh,role=execute]
----
hub-login user1
oc whoami
oc get nodes
----

== Accessing Developer Hub

. Open **Developer Hub** from the OCP Console application menu.
. Log in with the same workshop credentials.
. Browse the AI Computer Vision software template and catalog entities.

Developer Hub URL pattern:

[source,sh]
----
https://developer-hub.apps.{hub_domain}
----

== Accessing DevSpaces

. Open **DevSpaces** from the OCP Console application menu.
. Start a workspace with your workshop user credentials.
. Clone sample projects from your GitLab group (`ws-userN`).

== Verifying NeuroFace CV inference

Check the hub gateway inference endpoint:

[source,sh,role=execute]
----
neuroface-cv-status
----

Inspect Gateway API routes and Skupper interconnect:

[source,sh,role=execute]
----
neuroface-cv-traffic
----

== Exploring spoke clusters

Log in to east or west spokes and inspect edge workloads:

[source,sh,role=execute]
----
east-login user1
neuroface-spoke-status
----

[NOTE]
====
Spoke API URLs and domains are injected automatically when the pattern detects managed cluster registration.
====
