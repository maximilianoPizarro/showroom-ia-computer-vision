[.lead]
*This is the heart of the workshop.* NeuroFace is a multimodal AI application that performs real-time PPE safety detection, face recognition, and object detection — served as KServe InferenceServices on Red Hat OpenShift AI across a federated east/west edge fleet.

== What is NeuroFace?

NeuroFace combines multiple AI capabilities into a single application:

.NeuroFace capabilities
[cols="1,2,2",options="header"]
|===
| Capability | Technology | Description

| *PPE detection*
| YOLOv8 (KServe InferenceService)
| Detects hardhat, safety-vest, and goggles in images and webcam feeds

| *Face detection*
| OpenVINO Model Server (OVMS)
| Real-time face detection using the `face-detection-retail-0005` model

| *Object detection*
| YOLOv4-tiny (OpenCV DNN)
| Detects 80 COCO object classes (person, car, bottle, etc.)

| *Operator chat*
| LLM via MaaS API
| Natural language Q&A for operators monitoring the feed
|===

The NeuroFace frontend is built with Angular 17 and PatternFly 5. The backend is a FastAPI Python service that orchestrates inference calls to the model servers.

== Hands-on: open the NeuroFace UI

Navigate to the NeuroFace web interface:

[source,sh]
----
https://neuroface.apps.{hub_domain}
----

Log in with your workshop credentials. You will see the NeuroFace dashboard with tabs for:

* *PPE Safety* — upload an image or use your webcam for PPE detection
* *Face Detection* — real-time face recognition via OVMS
* *Object Detection* — general purpose 80-class COCO detection
* *Chat* — operator chat powered by LLM

== Hands-on: PPE detection demo

This is the flagship feature. Try it now:

. Click the *PPE Safety* tab
. Click *Upload Image* and select a photo of a person wearing (or not wearing) safety equipment
. Click *Detect*
. Observe the bounding boxes drawn around detected items:
** 🟢 *hardhat* — green bounding box
** 🔵 *safety-vest* — blue bounding box
** 🟡 *goggles* — yellow bounding box
** 🔴 *person* (no PPE) — red bounding box

Each detection includes a *confidence score* (0-100%) and a *class name*.

TIP: Try uploading different images — construction workers, lab technicians, or even yourself wearing a hard hat. The model was trained on real-world PPE scenarios.

== Hands-on: webcam detection

If your browser supports webcam access:

. Click the *PPE Safety* tab
. Click *Start Webcam*
. The model runs inference on each frame, drawing bounding boxes in real time
. Put on a hard hat or safety vest and watch the detections update live

NOTE: Webcam detection sends frames to the YOLO model on the spoke cluster via the Skupper interconnect. Latency depends on your network connection to the hub gateway.

== Hands-on: face detection with OVMS

. Click the *Face Detection* tab
. Upload an image with faces or start the webcam
. OVMS returns bounding boxes around detected faces using the `face-detection-retail-0005` model

The OVMS face detection model runs on the spoke cluster as a separate serving endpoint. It uses Intel OpenVINO optimized inference.

== Under the hood: YOLO PPE inference

The PPE detection pipeline works as follows:

. The frontend sends an image (base64-encoded or multipart) to the backend at `POST /api/ppe/detect`
. The backend forwards the image to the KServe InferenceService `yolo-ppe-serving`
. YOLO processes the image and returns JSON with detections:

[source,json]
----
{
  "detections": [
    {
      "class_name": "hardhat",
      "confidence": 0.94,
      "bbox": [120, 45, 280, 190]
    },
    {
      "class_name": "safety-vest",
      "confidence": 0.87,
      "bbox": [100, 200, 320, 480]
    }
  ],
  "inference_time_ms": 42,
  "model": "yolo-ppe-v1.5.0",
  "source_cluster": "east"
}
----

[start=4]
. The frontend draws bounding boxes on the original image using the bbox coordinates

== CV journey: east/west 50/50 traffic split

The NeuroFace CV journey federates PPE inference across both spokes. The hub gateway uses a Gateway API HTTPRoute with 50/50 weighted traffic distribution:

[source,yaml]
----
apiVersion: gateway.networking.k8s.io/v1
kind: HTTPRoute
metadata:
  name: neuroface-cv-route
spec:
  parentRefs:
    - name: neuroface-gateway
  rules:
    - backendRefs:
        - name: yolo-ppe-serving-east
          weight: 50
        - name: yolo-ppe-serving-west
          weight: 50
----

Verify the traffic split by hitting the health endpoint multiple times and observing which spoke responds:

[source,sh,role=execute]
----
hub-login user1
----

[source,sh,role=execute]
----
for i in $(seq 1 10); do
  curl -sk https://neuroface-cv.apps.{hub_domain}/health
  echo
done
----

You should see responses alternating between `east` and `west` source clusters.

== Model storage with MinIO

The YOLO PPE model weights (`best.pt`) are stored in MinIO on the hub cluster and synchronized to spokes via Skupper:

* *Bucket*: `s3://models/ppe-detection/model/best.pt`
* *Serving*: KServe mounts the model from the MinIO-backed storage URI
* *Cross-cluster*: Skupper exposes the MinIO service from hub to spokes as an `ExternalName` service

== PPE event pipeline (optional)

When PPE violations are detected, NeuroFace can produce events to a Kafka topic:

. Detection event → `cv.ppe.detections` Kafka topic (AMQ Streams)
. Camel K integration → filters violations (missing PPE)
. Mailpit → sends alert notifications

This pipeline is optional and depends on AMQ Streams being deployed on the spoke.

== What you learned

* NeuroFace is a multimodal AI application combining YOLO PPE detection, OVMS face detection, object detection, and LLM chat
* PPE detection identifies hardhats, safety vests, and goggles with confidence scores and bounding boxes
* KServe InferenceServices serve the YOLO model on spoke clusters
* The hub gateway distributes traffic 50/50 across east and west via Gateway API HTTPRoute
* You can call the inference API directly via curl or use the NeuroFace UI
* Model weights are stored in MinIO and synchronized across clusters via Skupper
