OpenShift AI on the hub provides MaaS model connections for NeuroFace chat and the workshop playground.

== Open the OpenShift AI dashboard

[source,sh]
----
https://rhods-dashboard-redhat-ods-applications.apps.%HUB_DOMAIN%
----

Log in with your workshop credentials.

== Explore maas-workshop

image::m03-ai-secret.png[AI Secret Flow,align=center,width=100%]


[source,sh,role=execute]
----
hub-login %USER_NAME%
oc get namespace maas-workshop
----

In the dashboard: *Data Science Projects* → *maas-workshop* → browse model connections (Llama Scout 17B, Granite 3.2, etc.).

== GitOps configuration

[source,yaml]
----
# charts/all/openshift-ai-hub/templates/all.yaml (excerpt)
apiVersion: dscinitialization.opendatahub.io/v2
kind: DSCInitialization
metadata:
  name: default-dsci
---
apiVersion: datasciencecluster.opendatahub.io/v2
kind: DataScienceCluster
metadata:
  name: default-dsc
spec:
  components:
    dashboard:
      managementState: Managed
    kserve:
      managementState: Managed
      defaultDeploymentMode: RawDeployment
----

[source,yaml]
----
# charts/all/openshift-ai-hub/values.yaml (excerpt)
maas:
  endpoint: "https://maas-rhdp.apps.maas.redhatworkshops.io"
  credentialsSecretName: openshift-ai-maas-credentials
  models:
    - id: llama-scout-17b
      displayName: Llama Scout 17B (workshop default)
----

== Verify KServe on spokes

[source,sh,role=execute]
----
east-login %USER_NAME%
oc get inferenceservice -n neuroface-cv
----

=== Check DataScienceCluster and InferenceService status

[source,sh,role=execute]
----
oc get datasciencecluster -n redhat-ods-applications -o custom-columns=NAME:.metadata.name,PHASE:.status.phase
oc get inferenceservice -n neuroface-cv -o custom-columns=NAME:.metadata.name,READY:.status.conditions[0].status
----

.Expected output
[source,text]
----
NAME          PHASE
default-dsc   Ready

NAME                    READY
yolo-ppe-serving        True
----

== What you learned

* OpenShift AI exposes MaaS models via `maas-workshop`
* DSCInitialization and DataScienceCluster CRs are GitOps-managed
* MaaS API keys flow from Vault → Secret → dashboard connection
