= Model Serving with OpenShift AI
:_content-type: PROCEDURE

++++
<span class="workshop-time-badge">⏱ ~15 min</span>
++++

Red Hat OpenShift AI (RHOAI) provides the ML platform layer for this pattern. It manages model serving runtimes, data science projects, and notebook workbenches. In this module you explore the OpenShift AI dashboard and inspect how KServe serves the YOLO PPE model.

== What is OpenShift AI?

OpenShift AI deploys a `DataScienceCluster` custom resource that installs:

* *KServe* — serverless model serving with autoscaling (HPA min 1, max 4)
* *Dashboard* — web UI for managing data science projects, models, and notebooks
* *Serving runtimes* — pre-configured runtimes for common ML frameworks
* *Data connections* — S3-compatible storage for model artifacts (MinIO in this pattern)
* *Workbenches* — Jupyter notebooks for model training and experimentation

== Architecture in this pattern

[cols="1,2",options="header"]
|===
| Cluster | OpenShift AI role

| *Hub*
| Full DataScienceCluster — dashboard, KServe, workbenches for PPE model training

| *East spoke*
| DataScienceCluster — KServe for inference (`yolo-ppe-serving` InferenceService)

| *West spoke*
| DataScienceCluster — KServe for inference (same as east)
|===

The operator subscription on spokes:

[source,yaml]
----
# values-east.yaml
subscriptions:
  rhods:
    name: rhods-operator
    namespace: redhat-ods-operator
    channel: stable-3.4
    source: redhat-operators
----

== Hands-on: explore the OpenShift AI dashboard

Open the OpenShift AI dashboard on the hub:

. Open the OCP Console on the hub cluster
. Click the application menu (grid icon) → *Red Hat OpenShift AI*
. Or navigate directly:

[source,sh]
----
https://rhods-dashboard-redhat-ods-applications.apps.{hub_domain}
----

In the dashboard, explore:

* *Data Science Projects* — namespaces configured for ML workloads
* *Model Serving* — deployed InferenceServices
* *Serving Runtimes* — available model serving backends

== Hands-on: inspect the KServe InferenceService

The YOLO PPE model is served as a KServe `InferenceService`. Check it on the east spoke:

[source,sh,role=execute]
----
east-login user1
----

[source,sh,role=execute]
----
oc get inferenceservice -n neuroface-cv -o wide
----

Inspect the InferenceService details:

[source,sh,role=execute]
----
oc get inferenceservice yolo-ppe-serving -n neuroface-cv -o yaml
----

Key fields to note:

* `spec.predictor.model.modelFormat` — the model framework
* `spec.predictor.model.storageUri` — where the model weights are loaded from (MinIO)
* `status.url` — the inference endpoint URL
* `status.conditions` — should all be `True`

== Hands-on: check serving pods

[source,sh,role=execute]
----
oc get pods -n neuroface-cv -l serving.kserve.io/inferenceservice=yolo-ppe-serving
----

The serving pod runs the pre-built container image `neuroface-ppe-serving:v1.5.0` which packages:

* YOLOv8 model with PPE-trained weights
* KServe v1 and v2 compatible API endpoints
* OpenCV headless for image preprocessing

== Model storage with MinIO

Models are stored in MinIO — an S3-compatible object store deployed on the hub cluster. The KServe `storageUri` points to the MinIO bucket:

[source,text]
----
s3://models/ppe-detection/model/best.pt
----

MinIO is accessible from spokes through Skupper Service Interconnect, which creates an `ExternalName` service that routes S3 traffic across the cluster boundary.

Verify MinIO is running on the hub:

[source,sh,role=execute]
----
hub-login user1
----

[source,sh,role=execute]
----
oc get pods -n industrial-edge-minio -l app=minio 2>/dev/null || echo "MinIO namespace may vary — check openshift-gitops applications"
----

== PPE workbench (optional)

The pattern can deploy a Jupyter notebook workbench (`ppe-workbench`) pre-loaded with a PPE detection notebook:

* Notebook: `ppe-detection.ipynb`
* Upload a `.jpg`, run YOLO inference, visualize bounding boxes in matplotlib
* Uses the same model weights as the production InferenceService

If the workbench is deployed, access it from the OpenShift AI dashboard under *Data Science Projects* → *neuroface*.

== What you learned

* OpenShift AI provides the ML platform with KServe, dashboard, and workbenches
* The YOLO PPE model is served as a KServe InferenceService with autoscaling
* Model weights are stored in MinIO and accessed via S3 storage URIs
* The DataScienceCluster CR manages all AI platform components
* Optional PPE workbenches allow interactive experimentation with the model
