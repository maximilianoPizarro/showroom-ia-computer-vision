NeuroFace chat uses MaaS (Models as a Service) — an external LLM endpoint. Obtain an API key and test inference with curl.

== MaaS configuration

image::m05-maas-inference.png[MaaS Inference Flow,align=center,width=100%]


[source,yaml]
----
# charts/all/spoke-neuroface/values.yaml (excerpt)
neuroface:
  chat:
    modelEndpoint: "https://maas-rhdp.apps.maas.redhatworkshops.io/v1"
    modelName: "llama-scout-17b"
----

== Obtain the API key

=== Option A — OpenShift AI dashboard

Open *maas-workshop* → Llama Scout 17B connection → copy API key.

=== Option B — Kubernetes Secret

[source,sh,role=execute]
----
hub-login %USER_NAME%
export API_KEY=$(oc get secret openshift-ai-maas-credentials -n maas-workshop \
  -o jsonpath='{.data.api-key}' 2>/dev/null | base64 -d)
echo "Key length: ${#API_KEY} characters"
----

IMPORTANT: Do not share or commit API keys. Keys are loaded from Vault during pattern installation.

== Test MaaS chat completion

[source,sh,role=execute]
----
export API_KEY="${API_KEY:-$(oc get secret openshift-ai-maas-credentials -n maas-workshop -o jsonpath='{.data.api-key}' 2>/dev/null | base64 -d)}"
curl -sk -H "Authorization: Bearer $API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"model":"llama-scout-17b","messages":[{"role":"user","content":"Explain PPE detection in one sentence"}]}' \
  https://maas-rhdp.apps.maas.redhatworkshops.io/v1/chat/completions | head -c 500
echo
----

.Expected output (truncated)
[source,json]
----
{
  "id": "chatcmpl-xxx",
  "object": "chat.completion",
  "choices": [{
    "message": {
      "role": "assistant",
      "content": "PPE detection uses computer vision..."
    }
  }]
}
----

== Test with different models

[source,sh,role=execute]
----
curl -sk -H "Authorization: Bearer $API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"model":"granite-3-2-8b-instruct","messages":[{"role":"user","content":"What is YOLO object detection?"}],"max_tokens":100}' \
  https://maas-rhdp.apps.maas.redhatworkshops.io/v1/chat/completions | python3 -m json.tool 2>/dev/null | head -20
----

.Expected output (truncated)
[source,json]
----
{
  "id": "chatcmpl-xxx",
  "object": "chat.completion",
  "choices": [{
    "message": {
      "role": "assistant",
      "content": "YOLO (You Only Look Once) is a real-time object detection..."
    }
  }]
}
----

== Test NeuroFace gateway

[source,sh,role=execute]
----
curl -sk -o /dev/null -w "NeuroFace gateway: HTTP %{http_code}\n" \
  https://neuroface.apps.%HUB_DOMAIN%/
----

.Expected output
[source,text]
----
NeuroFace gateway: HTTP 200
----

== What you learned

* MaaS provides external LLM inference for NeuroFace chat
* API keys flow from Vault → Secret → dashboard and backend
* curl validates end-to-end MaaS and gateway connectivity
