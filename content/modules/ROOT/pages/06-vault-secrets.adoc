Secrets in this pattern flow from HashiCorp Vault through External Secrets Operator (ESO) into Kubernetes Secrets. No credentials are committed to Git.

image::m03-ai-secret.png[Vault Secret Injection Flow,align=center,width=100%]

== How the secrets pipeline works

[cols="1,3",options="header"]
|===
| Step | What happens

| 1. Template
| `values-secret.yaml.template` defines secret names and fields. Fields with `onMissingValue: generate` are auto-created by Vault.

| 2. Vault load
| `make load-secrets` (CLI) or manual `vault kv put` (console install) stores secrets in Vault at `secret/hub/<name>`.

| 3. ESO sync
| `ExternalSecret` CRs watch Vault paths via `ClusterSecretStore vault-backend` and project them into Kubernetes Secrets.

| 4. Pod consumption
| Deployments mount the K8s Secret as environment variables or volumes.
|===

== Inspect a specific secret

[source,sh,role=execute]
----
oc exec vault-0 -n vault -- vault kv get -format=json secret/hub/rhbk-credentials 2>/dev/null | python3 -c "
import sys, json
d = json.load(sys.stdin)
data = d.get('data', {}).get('data', {})
for k in data:
    print(f'  {k}: {data[k][:8]}...' if len(str(data[k])) > 8 else f'  {k}: {data[k]}')
"
----

TIP: Vault stores secrets at `secret/data/hub/<name>` (KV v2 adds the `data/` prefix automatically).

== The values-secret.yaml.template

This file defines all secrets the pattern needs. Fields with `onMissingValue: generate` are auto-created:

[source,yaml]
----
# values-secret.yaml.template
version: "2.0"

secrets:
  - name: gitlab-credentials          # <1>
    fields:
      - name: root-password
        onMissingValue: generate       # <2>
        vaultPolicy: validatedPatternDefaultPolicy
      - name: runner-token
        onMissingValue: generate

  - name: rhbk-credentials            # <3>
    fields:
      - name: admin-password
        onMissingValue: generate
      - name: db-password
        onMissingValue: generate

  - name: developer-hub-secrets
    fields:
      - name: session-secret
        onMissingValue: generate
      - name: gitlab-token             # <4>
        value: "set after GitLab deploys"

  - name: maas-credentials             # <5>
    fields:
      - name: api-key
        value: "your MaaS API key"

  - name: spoke-credentials            # <6>
    fields:
      - name: east-token
        value: "oc create token output"
      - name: east-api-url
        value: "https://api.<east>:6443"
----
<1> GitLab admin password and CI runner token
<2> `generate` means Vault creates a random value at install time
<3> Keycloak admin and PostgreSQL passwords
<4> Manual: set after GitLab deploys with a PAT (`api` scope)
<5> Optional: only needed for MaaS LLM chat in NeuroFace
<6> Only for `spokeCredentials.mode=vault` (inline mode uses Pattern CR extraParameters instead)

== Check ExternalSecret sync status

ESO projects Vault secrets into K8s Secrets:

[source,sh,role=execute]
----
oc get externalsecret -A
----

.Expected output
[source,text]
----
NAMESPACE         NAME                  STORE           STATUS         READY
keycloak-system   keycloak-admin-user   vault-backend   SecretSynced   True
keycloak-system   postgresql-db         vault-backend   SecretSynced   True
----

If any shows `SecretSyncedError`, the Vault path may be empty. Fix with:

[source,sh,role=execute]
----
oc exec vault-0 -n vault -- vault kv put secret/hub/rhbk-credentials \
  admin-password="$(openssl rand -base64 16)" \
  db-password="$(openssl rand -base64 16)"
----

Then force ESO to re-sync:

[source,sh,role=execute]
----
oc annotate externalsecret -n keycloak-system --all force-sync=$(date +%s) --overwrite
----

== Console install vs CLI install

[cols="2,3,3",options="header"]
|===
| Method | Secrets loading | Action needed

| `./pattern.sh make install`
| `make load-secrets` runs automatically
| Fill `~/values-secret-ia-computer-vision.yaml` before install

| Pattern CR (OCP Console)
| Secrets are NOT loaded
| After Vault initializes (wave 2), load secrets manually with `vault kv put`
|===

Manual load commands for console installs:

[source,sh,role=execute]
----
oc exec vault-0 -n vault -- vault kv put secret/hub/gitlab-credentials \
  root-password="$(openssl rand -base64 16)" runner-token="$(openssl rand -base64 16)"

oc exec vault-0 -n vault -- vault kv put secret/hub/rhbk-credentials \
  admin-password="$(openssl rand -base64 16)" db-password="$(openssl rand -base64 16)"

oc exec vault-0 -n vault -- vault kv put secret/hub/developer-hub-secrets \
  session-secret="$(openssl rand -base64 32)" gitlab-token="placeholder"

oc exec vault-0 -n vault -- vault kv put secret/hub/maas-credentials \
  api-key="your-maas-key-here"
----

== Verify the full chain

End-to-end check: Vault → ESO → K8s Secret → Pod:

[source,sh,role=execute]
----
echo "=== Vault ==="
oc exec vault-0 -n vault -- vault kv list secret/hub 2>/dev/null

echo "=== ExternalSecrets ==="
oc get externalsecret -A --no-headers 2>/dev/null | awk '{printf "%-20s %-25s %s\n", $1, $2, $6}'

echo "=== K8s Secrets (from Vault) ==="
oc get secret -n keycloak-system -l externalsecrets.io/owner=ExternalSecret --no-headers 2>/dev/null
----

== What you learned

* Secrets flow: `values-secret.yaml.template` → Vault → ESO `ExternalSecret` → K8s Secret → Pod
* `onMissingValue: generate` auto-creates passwords at install time
* Console installs require manual `vault kv put` after Vault initializes
* `oc annotate externalsecret --all force-sync=...` forces ESO to re-read from Vault
