== Ideas for customization

Now that you understand the platform, here are ideas to extend it:

=== Add GPU acceleration

The YOLO PPE model currently runs on CPU. To enable GPU inference:

. Add a GPU-enabled node pool to a spoke cluster (NVIDIA T4/A10G)
. Install the NVIDIA GPU Operator
. Update the KServe InferenceService to request GPU resources:

[source,yaml]
----
spec:
  predictor:
    model:
      resources:
        limits:
          nvidia.com/gpu: "1"
----

=== Add a third spoke cluster

Scaling the fleet is straightforward:

. Add a new entry to `managedClusters` in `values-hub.yaml`
. Create a `values-south.yaml` file (copy `values-east.yaml` and adjust names)
. Update the Gateway API HTTPRoute weights (e.g., 33/33/34)
. Push to Git — ArgoCD handles the rest

=== Train a custom YOLO model

The PPE model can be retrained with your own dataset:

. Open the `ppe-retrain-workbench` notebook in OpenShift AI
. Upload your labeled images to MinIO
. Train with Ultralytics YOLOv8: `yolo train data=custom.yaml model=yolov8n.pt`
. Export the new `best.pt` to MinIO
. KServe picks up the new model on the next pod restart

=== Add new detection classes

The current model detects hardhat, safety-vest, and goggles. You can extend it to detect:

* Safety boots
* High-visibility clothing
* Ear protection
* Face shields
* Gloves

Retrain the model with annotated images for the new classes and update the NeuroFace backend to display them.
